## Repo Map

### Quick start (where to begin)
- **Start here:** `README.md`
- **Main entry point:** `path/to/main/file` (ex: `src/index.js` or `main.py`)
- **How to run:** `docs/SETUP.md` (or put run steps in this README)

### Key folders (what lives where)
- `src/` — main application/source code
- `docs/` — documentation (setup, notes, guides)
- `tests/` — automated tests
- `scripts/` — helper scripts (build, deploy, data tools)
- `assets/` — images, audio, static files
- `data/` — datasets or sample data (if included)

### Key files (important single files)
- `README.md` — overview + how to use this repo
- `LICENSE` — usage permissions
- `.gitignore` — files Git won’t track
- `package.json` / `requirements.txt` — dependencies
- `.env.example` — environment variables template (don’t commit real secrets)

### Where to make changes (common tasks)
- Change the app logic: `src/`
- Change documentation: `README.md` or `docs/`
- Add a feature: create a new file in `src/` and link it here
- Fix a bug: find the related module in `src/` then update tests in `tests/`
